🧠 One-liner: Like looking up a word in a paper dictionary — you don't start on page 1, you flip to the middle and narrow it down from there.

🏠 The Big Picture: Imagine you have a phone book sorted by last name, and you're looking for "Zhang." You wouldn't read every name one by one — that'd take forever. You'd flip to roughly the middle, see if you're too early or too late, then throw away the half that doesn't contain "Zhang," and repeat with the remaining half. In just a few flips, you land on the right page. This code does exactly that for a sorted list of numbers — it keeps cutting the search area in half until it finds the target or knows it's not there.

👣 Walkthrough:
1. Mark the two ends of the shelf (left = first slot, right = last slot).
   → In code: left = 0, right = len(arr) - 1
2. While there's still space to search between left and right:
   → In real life: as long as the two ends haven't crossed, the target could still be in there.
3. Pick the middle slot: mid = (left + right) // 2.
   → In real life: like grabbing the middle page of the remaining book section.
4. Peek at what's in that middle slot:
   - If it's exactly the target → found it! Return the slot number.
   - If the middle item is too small → throw away the left half (left = mid + 1).
   - If the middle item is too big → throw away the right half (right = mid - 1).
   → In real life: "I see 'Liu' here — 'Zhang' must be further back, so I'll tear off the left half and keep going."
5. If the space is empty (left crossed right) → return -1, meaning "not on the shelf."

📝 Summary: This is "the dictionary lookup method" — it takes a sorted list and a target, keeps splitting the search range in half, and either returns the position or signals "not found."
