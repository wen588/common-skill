这段代码好比你在翻一本厚厚的电话本找某个人的名字。

你肯定不会从第一页翻到最后一页——那样太慢了。你会先猜到中间位置，翻开一看：
- 如果这一页正好就是你要找的人 → 找到了
- 如果这一页的姓氏比目标小（比如你找"张"，翻到"刘"）→ 说明目标在后半本，扔掉前半本
- 如果这一页的姓氏比目标大（翻到"赵"）→ 说明目标在前半本，扔掉后半本

每次把剩下的书对半切开，只保留可能含目标的那一半。一直重复，直到找到或者确定没有。

拆开看代码的每一步：
1. left/right = 0 / len(arr)-1 → 画出一个搜索范围，好比用两根手指捏住书的两端
2. while left <= right → 只要两根手指还没交叉（还有书页可翻），就继续找
3. mid = (left + right) // 2 → 找到当前范围的中间位置，就像每次对准书的中间翻开
4. if arr[mid] == target → 翻到的这一页正好是你要的
5. elif arr[mid] < target → 中间页太小了，把左手指移到 mid+1（扔掉前半本）
6. else → 中间页太大了，把右手指移到 mid-1（扔掉后半本）
7. return -1 → 两根手指交叉了还没找到，说明这本电话本里根本没有这个人

一句话总结：每次把搜索范围砍掉一半，飞速定位目标——这就是"二分查找"。
